## CONTEXT
You're debugging a sports betting prediction system (Google Apps Script). The issue: **Quarter Over/Under picks are generated in OU_Log but don't appear on Bet_Slips**. 

Two problems:
1. **Config_Accumulator missing OU settings** - gets auto-regenerated, removing manual edits
2. **Quarter book lines (Q1-Q4 columns in UpcomingClean) not being read** - accumulator only reads FT line

## FILES TO MODIFY
- `docs/Contract_Enforcement.gs` (accumulator module ~line 2800-3200)

## THE BUGS

### Bug 1: Config_Accumulator Auto-Regeneration
The `_canonicalizeAccumulatorConfig_()` function rewrites Config_Accumulator on every run, deleting custom keys.

**Fix needed:**
Add OU-related defaults to the canonicalization function so they persist:

```javascript
// Find _canonicalizeAccumulatorConfig_() around line 2850
// In the defaults object, ADD these entries:

includeOUSignals: true,
ouMinConf: 50,
ouMinEdge: 2.0,
hq_min_confidence: 55,
hq_skip_ties: true,
includeHQSignals: true
```

### Bug 2: Quarter Book Lines Not Read
The `buildAccumulator()` function only reads the `FT` column for O/U book lines. It needs to also read `Q1`, `Q2`, `Q3`, `Q4` columns from UpcomingClean.

**Current code** (around line 3050):
```javascript
const ftBookLine = upcomingRow.ft ? parseFloat(upcomingRow.ft) : null;
```

**Needs to become:**
```javascript
const ftBookLine = upcomingRow.ft ? parseFloat(upcomingRow.ft) : null;
const q1BookLine = upcomingRow.q1 ? parseFloat(upcomingRow.q1) : null;
const q2BookLine = upcomingRow.q2 ? parseFloat(upcomingRow.q2) : null;
const q3BookLine = upcomingRow.q3 ? parseFloat(upcomingRow.q3) : null;
const q4BookLine = upcomingRow.q4 ? parseFloat(upcomingRow.q4) : null;
```

### Bug 3: OU Candidate Building Ignores Quarters
In `_loadTier2Signals()` or similar function that builds OU candidates, quarter picks from OU_Log are parsed but their book lines aren't matched against UpcomingClean's Q1-Q4 columns.

**Search for:** `ouCandidates` or `_loadTier2Signals` around line 2900-3100

**Fix pattern:**
When processing OU_Log entries like "Q1: Over 52.5", extract:
- Quarter: `Q1`
- Direction: `Over`
- Predicted line: `52.5`

Then match against `q1BookLine` from UpcomingClean (not just `ftBookLine`).
